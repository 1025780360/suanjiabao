# 服装成本管理系统项目规划

## 1. 产品定位

本项目第一阶段不是传统 ERP，也不是要求用户先建立完整资料库的管理系统，而是一款面向广州服装档口老板的极简算价工具。

核心表达：

> 不用懂系统，拍个款、填几个数，马上知道这件衣服卖多少钱才不亏。

第一阶段产品形态应该像手机计算器，而不是 ERP。用户先把价格算起来，系统在背后自动沉淀款式、报价、成本和客户记录。等使用频率起来以后，再逐步长成管理系统。

产品核心方向：

> 把传统档口老板的手算成本，变成手机上 1 分钟完成的报价。

## 2. 目标用户

- 服装工厂老板、业务员、跟单员、采购员、财务人员
- 服装档口经营者
- 小型服装贸易公司或供应链服务商

## 3. 核心业务目标

- 快速拍照或上传款式图。
- 输入少量关键数字，立即得到单件成本、建议卖价和最低接单价。
- 自动提示低价接单风险。
- 自动计算 50 件、100 件、300 件、500 件的预计利润。
- 自动保存历史算价记录，下次类似款可以复制。
- 后台悄悄沉淀款式库、报价库、成本库和客户记录。
- 后续再扩展供应商、物料库、BOM、权限、报表和 SaaS 多租户。

## 5. 成本计算范围

单件成本在内部仍然要完整，但前端只展示老板听得懂的结果。

内部计算拆分为：

- 面料成本
- 辅料成本
- 工序加工费
- 包装成本
- 物流成本
- 其他成本
- 税费
- 损耗
- 利润

建议批发价 = 单件总成本 + 期望利润。

最低接单价 = 单件总成本 + 基础安全利润。

当用户输入的报价低于最低接单价时，系统直接提示风险。

第一版优先支持：

- 物料损耗率
- 工序单价
- 辅料包装估算
- 运费估算
- 返工风险预留
- 档口租金和人工摊销
- 固定利润或利润率
- 不同订单数量的预计利润

颜色、尺码、尺码配比、阶梯报价、汇率、税率后续再做。

## 6. 技术架构

### 6.1 前端

- Vue 3
- TypeScript
- Vite
- Pinia
- Vue Router
- Element Plus
- Axios
- ECharts

前端第一版以手机浏览器使用为主，电脑端作为后台查看和运营管理入口。

### 6.2 后端

- Python 3.10+
- Django
- Django REST Framework
- django-cors-headers
- PostgreSQL 或 MySQL
- Redis，可用于缓存和异步任务
- Celery，可用于后续 Excel 导入、报表生成等耗时任务

### 6.3 部署

优先评估 Zeabur：

- 前端作为静态站点部署
- 后端作为 Django 服务部署
- 数据库使用 Zeabur PostgreSQL 或外部云数据库
- 媒体文件后续可接入对象存储

如果 Zeabur 对 Django 长期运行、数据库备份或文件存储不够方便，再切换到云服务器 + Docker Compose。

## 7. 后端应用划分

建议 Django 后端按业务拆分应用：

- `accounts`：用户、企业、角色、权限
- `base`：公共字典、单位、币种、税率
- `suppliers`：供应商资料
- `materials`：面料、辅料、价格记录
- `styles`：款式、颜色、尺码、款式图片
- `processes`：工艺工序和加工费
- `bom`：BOM 清单
- `costing`：成本核算、成本版本、报价
- `orders`：订单与订单成本追踪
- `reports`：报表统计
- `imports`：Excel 导入导出任务

## 9.1 账号数据隔离规则

商业化版本必须做到不同账号之间的数据互相不可见。第一阶段按以下规则实现：

- 所有用户创建的款式、报价、成本、客户、品类模板都必须绑定当前登录账号。
- 后端数据模型必须同时保留 `tenant` 和 `created_by`。
- 普通用户查询数据时，只能返回 `tenant = 当前企业` 且 `created_by = 当前用户` 的记录。
- 企业管理员后续可以增加“查看本企业全部员工数据”的权限，但不能默认开放。
- 前端只做展示过滤，真正的数据隔离必须由 Django API 层完成。
- 自定义品类属于创建它的账号，其他账号默认不可见。
- 系统内置品类模板可以所有账号共用，但用户修改后的模板必须另存为自己的模板。

## 11. 第一版成功标准

- 用户打开手机后，1 分钟内能完成一次算价。
- 不要求用户先录供应商、物料库、BOM 或工序库。
- 用户能看到单件成本、建议卖价、最低接单价和不同数量利润。
- 低于安全价时有明确风险提醒。
- 每次算价自动保存为历史款式。
- 类似款可以复制后改数字。
- 电脑端能查看历史款式和报价。
- 系统可部署到 Zeabur 并稳定访问。
